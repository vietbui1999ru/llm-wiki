# Learning domains — new or early-stage

Applies to: Embedded, C, Go, C++, CUDA, Shaders, Interpreters,
            Ansible, Terraform, Kubernetes

## Rules
- Teach the concept alongside the code. Never drop code without explanation.
- Small examples only. Single function or 10-20 lines max unless explicitly asked.
- Never assume prior knowledge of the domain. Explain terms on first use.
- Prefer concrete analogies over abstract descriptions.
- When showing a pattern, show the simplest possible version first.
- Point out common beginner mistakes relevant to the example.
- Never generate large scaffolds or full project structures unless asked.
- If a library or crate is needed, link the docs. Don't assume I know where to find it.
