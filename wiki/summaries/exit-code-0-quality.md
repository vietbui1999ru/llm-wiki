# Exit Code 0 Is Not Quality: 198 Agents, 30 Campaigns

Solo developer building a 668K-line TypeScript platform documents a production multi-agent orchestration system that independently converges with Google DeepMind's December 2025 scaling study — and extends it.

## The Core Failure

Autonomous agent completed multi-phase feature (Show Bible entity cards), passed typecheck, exited confidently. 37 of 38 entities were invisible. Structurally correct, completely non-functional.

> "Exit code 0 means no errors — not it's good."

This failure built the entire verification system.

## The Architecture: Four-Tier Strict Hierarchy

```
Fleet (session orchestrator)
  └─ Archon (campaign manager, amnesiac by design)
       └─ Marshals (wave executors)
            └─ Skills (task primitives)
```

**Data flow is strictly top-down.** No lateral agent-to-agent communication at the same level. This is a star topology — adding agent N doesn't increase coordination complexity for agents 1 through N-1.

Agrees with DeepMind: unstructured "bags of agents" amplify errors 17.2x. Coordination failures = 36.9% of all failures. Benefits plateau at ~4 concurrent agents.

## Persistence: The Campaign File

Archon is **amnesiac by design**. Everything externalized:
- Personality: profile file
- Knowledge: capability manifests
- Current work: campaign file (feature ledgers, decision logs, phase tracking, architectural constraints)

When context fills, Archon writes state to campaign file and exits. Next invocation reads and resumes. The campaign file is the agent's brain. This enables multi-day campaigns that survive context window boundaries.

No other surveyed framework (LangGraph, CrewAI, AutoGen, Devin, Cursor) implements persistent campaign state across sessions.

## Parallel Execution Design

- Each agent gets strict directory scope + separate git worktree
- Agents literally cannot touch each other's files
- **Discovery relay**: compresses findings from Wave N, injects into Wave N+1's context — agents don't reinvent each other's decisions
- **Scope overlap detection**: `src/domains/sandbox/` overlaps with `src/domains/sandbox/dialogue/` but not with `src/domains/studio/`
- **Circuit breaker**: 1 conflict = pause wave; 2 conflicts = stop session

## The Five Protocol Rules (from failures)

1. **Layer, don't replace** — verify before removing working code
2. **Design target, not delta** — define end state before starting
3. **Plan validation gate** — check decomposed plan against original request before execution
4. **Merge-before-cleanup** — worktree branches merged before cleanup
5. **Claim-before-resume ordering** — prevent TOCTOU race on campaign files

## The Taste Layer

Archon's profile encodes the developer's aesthetic judgment as ranked decision heuristics and a **Three Words test**: Inhabited, Responsive, Intentional. Applied at every visual/experiential decision point. Anti-patterns explicitly encoded: no centered-heading-CTA hero sections, no card grids with hover scale.

**This cannot be abstracted into a framework.** It requires deep project knowledge. The taste layer is the strongest differentiator and the hardest limitation simultaneously.

## Documentation as the Multiplier

278,881 lines across 911 markdown files (~42% of codebase size). Without it, agents guess. With it, they operate from institutional knowledge.

The final proof of concept: this very case study was generated by 5 parallel agents, worktree-isolated, using the discovery relay — the system documented itself.
